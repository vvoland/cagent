Remote MCP Servers

Connect cagent to cloud services via remote MCP servers with built-in OAuth authentication.

Overview

cagent supports connecting to remote MCP servers over SSE (Server-Sent Events) and Streamable HTTP transports. Many popular services offer MCP endpoints with OAuth ‚Äî cagent handles the authentication flow automatically.

toolsets:
  - type: mcp
    remote:
      url: "https://mcp.linear.app/sse"
      transport_type: "sse"
üí° OAuth flow

When you connect to a remote MCP server that requires OAuth, cagent opens your browser automatically for authentication. Tokens are cached for subsequent sessions.

Configuration

toolsets:
  - type: mcp
    remote:
      url: "https://mcp.example.com/sse"
      transport_type: "sse"          # or "streamable"
      headers:
        Authorization: "Bearer token" # optional: static auth

For full configuration details, see the Tool Config page.

Example: Multi-Service Agent

Combine multiple remote MCP servers in a single agent:

agents:
  root:
    model: anthropic/claude-sonnet-4-0
    instruction: |
      You help manage projects and deployments.
    toolsets:
      - type: mcp
        remote:
          url: "https://mcp.linear.app/sse"
          transport_type: "sse"
        instruction: Use Linear for issue tracking.
      - type: mcp
        remote:
          url: "https://api.githubcopilot.com/mcp"
          transport_type: "sse"
        instruction: Use GitHub for code and PRs.
      - type: mcp
        remote:
          url: "https://mcp.vercel.com/"
          transport_type: "sse"
        instruction: Use Vercel for deployments.
