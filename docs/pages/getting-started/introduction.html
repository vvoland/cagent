Why cagent?

After spending years building AI agents using various frameworks, the Docker team kept asking the same questions:

  • How do we make building agents less of a hassle? â€” Most agents use the same building blocks. cagent provides them out of the box.
  • Can we reuse those building blocks? â€” Declarative YAML configs mean you can mix and match agents, models, and tools without rewriting code.
  • How can we share agents easily? â€” Push agents to any OCI registry and run them anywhere with a single command.

How it Works

At its core, cagent follows a simple loop:

  1. You define agents in YAML â€” their model, instructions, tools, and sub-agents.
  2. You run an agent via the TUI, CLI, or API.
  3. The agent processes your request â€” calling tools, delegating to sub-agents, and reasoning step by step.
  4. Results stream back in real-time via an event-driven architecture.
# A minimal agent definition
agents:
  root:
    model: openai/gpt-4o
    description: A helpful assistant
    instruction: You are a helpful assistant.
    toolsets:
      - type: think
# Run it
$ cagent run agent.yaml
